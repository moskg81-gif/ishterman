# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

**ИшТап (IshTap)** — двуязычный (русский/кыргызский) маркетплейс задач для Кыргызстана. Заказчики публикуют платные задания, исполнители откликаются. Всё приложение находится в одном файле: `ishtap_app.tsx`.

Нет системы сборки, package.json или бэкенда. Файл — самодостаточный React-компонент для запуска в Vite-проекте. Все данные сохраняются в `localStorage`.

## Architecture

### Single-file structure
`ishtap_app.tsx` содержит по порядку:
1. **Constants** — `LANGS`, `CITIES` (58 населённых пунктов Кыргызстана), `CATS` (18 категорий), строки перевода (`T`), CSS-анимации (`css`), ключи localStorage (`SK_*`)
2. **Storage helpers** — `load*` / `save*` функции обёртки над `localStorage`
3. **Seed data** — `SEED_TASKS` и `buildInitTasks()` для демо-заданий, `MOCK_OWNER_ID` / `DEMO_EXEC_ID` для тестовых пользователей
4. **Shared UI components** — `StarRating`, `Stars`, `Pill`, `SectionCard`, `BackBtn`, `ScreenHeader`, `StepBar`, `Avatar`, `ReviewBtn`
5. **Screen components** — по одному на каждый экран (`LoginScreen`, `TaskDetailScreen`, `TabCreate` и др.)
6. **Root `App` component** — хранит всё состояние, предоставляет контекст

### State management
Всё состояние живёт в корневом `App` и передаётся через единый React Context (`Ctx`), доступный через `useApp()`. Ключевые состояния:
- `tasks` / `responses` / `reviews` — сохраняются в localStorage при каждом обновлении через сеттеры-обёртки (`setTasksPersist` и др.)
- `allNotifs` — глобальная таблица уведомлений `[{id, userId, text, read, time}]`; фильтруется по пользователю при рендере
- `navStack` — императивный стек навигации; `push(entry)` / `pop()` управляют переходами
- `tab` — активная нижняя вкладка (`"tasks"`, `"create"`, `"profile"`)

### Navigation model
- Три нижних вкладки; исполнители видят только `tasks` + `profile` (без `create`)
- `push({type, ...props})` — добавляет экран поверх вкладки; строка `type` маппится на компонент в `renderNav()`
- `pop()` — возврат на предыдущий экран или активную вкладку
- `switchTab(id)` — сбрасывает стек и переключает вкладку

### Authentication flow
`authState`: `null` (загрузка) → `"login"` → `"role"` (если роль не выбрана) → `"app"`

### Roles
- **Заказчик (`customer`)**: публикует задания, выбирает исполнителей, завершает задания, оставляет отзывы
- **Исполнитель (`executor`)**: просматривает задания, отправляет отклики с ценой, получает отзывы

### Bilingual text
Все строки интерфейса в объекте `T` с ключами `ru` и `ky`. Доступ через `t = T[lang]`. Заголовки и описания заданий хранятся в полях `ru`/`ky` и `dRu`/`dKy`.

### Payment simulation
Приложение сейчас **бесплатное** — PaymentScreen не вызывается. Задание публикуется и отклик отправляется без оплаты. `PaymentScreen` оставлен как мёртвый код для будущего.

### Task lifecycle
`open` → `closed_by_choice` (исполнитель выбран) → `completed`
`open` → `cancelled` (заказчик отменил вручную)
`open` → `closed_expired` (TTL 7 дней, проверяется каждые 60 сек)
Реактивация (`closed_by_choice` → `open`) — максимум 3 раза на задание.

Отменённые задания (`cancelled`) не отображаются в ленте. При отмене всем откликнувшимся исполнителям приходит уведомление.

### CSS
Инжектируется через `<style>{css}</style>`. Использует CSS custom properties для тёмного режима (`--color-background-primary` и др.), включается добавлением `.dark-mode` к корневому div. Классы анимаций: `.fe` (fadeUp), `.si` (slideIn), `.tc` (tabSlide), `.pi` (popIn).

## Categories (CATS)

## UI / Design decisions

### Цвета
- Верхняя панель (TopBar): белая (`var(--color-background-primary)`)
- Нижний таб-бар: небесно-синий `#DAEEFF`
- Активная вкладка: янтарный `#EF9F27`
- Неактивная вкладка: `#85B7EB`
- Подписи всех вкладок: всегда `#EF9F27` (оранжевый)
- Заголовки экранов (ScreenHeader): `#DAEEFF` фон, текст `#0C447C`
- Поля поиска и выбора города: заливка `rgba(180,220,255,0.35)`, рамка `rgba(130,190,255,0.5)`

### Иконки таб-бара
- "Задачи": `/задачи.PNG` (30×30px)
- "Профиль": `/Профиль.PNG` (36×36px)
- "Создать": зелёная кнопка FAB с иконкой «+»
- Иконки находятся в папке `/public/`

### TopBar
- Геолокация ("Рядом") удалена полностью (кнопка и функция `handleGeo`)
- Поле города — такой же ширины как поле поиска
- Строки `useGeo` и `locating` в объекте `T` остались как мёртвый код (не удалять, не вредят)

### Иконки категорий — 3D стиль
- Форма: скруглённый квадрат `border-radius: 14px`, размер `44×44px`
- Фон: `linear-gradient(145deg, light, color)` — уникальный для каждой категории
- Тень: `box-shadow: 0 5px 0 shadow, 0 8px 16px color66` — создаёт эффект 3D кнопки
- Иконка: Tabler Icons (webfont), `font-size: 22px`, цвет `#fff`
- Библиотека подключена в `index.html`: `@tabler/icons-webfont@3.19.0`
- При нажатии: `transform: translateY(3px)` — эффект вдавливания

### Анимация складывания категорий
При выборе категории остальные иконки складываются влево (`width: 0, opacity: 0, transition: 0.35s`). Кнопка "Все" всегда остаётся видимой. Нажатие на "Все" возвращает все иконки.

### Мобильный viewport
`index.html` содержит `interactive-widget=resizes-visual` — предотвращает дёрганье экрана при открытии клавиатуры.

## Features implemented

### Фото при создании задания
- Только галерея (камера убрана — `capture="environment"` вызывала перезагрузку страницы на Android)
- До 3 фотографий, компрессия через `FileReader` + `canvas.toDataURL("image/jpeg", 0.7)`, максимум 1024px
- Кнопка ✕ на каждом фото + кнопка "Очистить все фотографии"

### Отмена задания (заказчик)
- В "Мои задания": кнопка "Отменить задание" для заданий со статусом `open`
- Предупреждение: "При отмене задания деньги не возвращаются"
- После отмены: статус → `cancelled`, задание пропадает из ленты
- Всем откликнувшимся исполнителям приходит уведомление об отмене

### Удаление заданий (заказчик)
- Завершённые (`completed`): кнопка "🗑 Удалить завершённое задание"
- Отменённые (`cancelled`): кнопка "🗑 Удалить отменённое задание"

### Удаление откликов (исполнитель)
- В "Мои отклики": кнопка "🗑 Удалить отклик" на каждом отклике (любой статус)
